## 1.1 Using Go for web applications
go作为比较新的语言，在Large-scale web applications开发领域越来越流行。开发Large-scale web applications需要具备如下特性：
> - Scalable (可扩展性)
> - Modular （模块化）
> - Maintainable （可维护性）
> - High-performance （高性能）

#### 1.1.1 Scalable web applications and Go
  web服务一般都需要具备可伸缩性，这意味我们开发的web应用能够简单和快速的提高响应大量请求的能力；web系统也需要能够通过增加硬件资源来实现线性扩展。

  可从以下两方面提高web服务的能力：
  > - `垂直扩展` : 通过提高单台机器的配置，比如增加cpu性能 ，提高内存空间等。机器性能提高了web服务自然也就可以跑的更快了。
  > - `水平扩展` : 通过增加机器的数量。可以将一个web服务的多个实例跑在多台机器上，将大量的请求平均分配到每个机器的服务去处理而不是全部集中在一台机器，可以有效的降低每个服务的压力。

  go语言完美的支持并发，go通过协程来实现并发，协程相对与进程和线程更加的轻量级，当处理大量请求时可以通过增加大量的协程来提高处理请求的速度。

  go语言相对与其他语言来说对环境的依赖更小，go编写的web服务可以编译为一个可执行文件，可执行可以在没有go环境的系统运行，这样就可以方便快速的将go编写的web服务部署到多台机器中。

#### 1.1.2 Modular web applications and Go
虽然go语言是静态语言，但是go的接口能够实现动态类型的效果，go语言的接口是一系列方法的集合，表示了一种行为。只要一种类型实现了接口的方法，那该类型也就成为了接口类型。这一机制意味着我们可以很容易继承已有的模块，通过实现模块的接口来实现自定义的功能，而又不需要改变模块的代码实现。

go也经常用来开发微服务，我们可以将一个大的服务按照功能拆解为独立的微服务，每一个微服务就是一个模块，我们可以对模块进行组合和复用来实现一个功能完整的web系统。

#### 1.1.3 Maintainable web applications and Go
go语言拥有清晰和简单的语法，而且go拥有很强大的一系列工具来帮助提高开发体验和写出具有可读性的代码。比如`gofmt`能够标准化代码格式，`godoc`能够根据代码的注释来生成api文档。`gotest`工具能够自动发现元源代码目录下的测试代码并执行测试。

#### 1.1.4 High performing web applications and Go
高可用性意味着服务需要在很短的时间内同时处理大量的请求，并能够快速的处理客户端的响应。而go的设计目标就是接近C的性能。go的编译速度很快，能够将源代码编译为一个可执行的文件，相比与通过解释器执行的语言性能又很大提升。go语言的一个重要的特性是支持协程高并发，高并发能够实现同一时间处理大量的请求。
